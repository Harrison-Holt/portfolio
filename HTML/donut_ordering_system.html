Donut Ordering System (AWS Serverless)

Description:
The Donut Ordering System is a fully serverless application built to process donut orders using AWS services. It supports real-time order submission, wait time estimation, receipt generation, and customer email confirmations using an event-driven architecture.

My Design Process

The system was designed for scalability and modularity, with a clear focus on using managed services to minimize infrastructure overhead. I used diagrams and flowcharts to plan each service interaction before building and deploying.

Technology Stack

  • AWS Lambda: Handles business logic for order processing and email generation.
  • API Gateway: Accepts HTTP requests from clients.
  • DynamoDB: Stores order data and inventory.
  • Step Functions: Manages the order workflow including wait time calculation and confirmation.
  • S3: Hosts downloadable order receipts.
  • SES (Simple Email Service): Sends order confirmation emails with pre-signed receipt URLs.
  • CodePipeline + CodeBuild + CodeDeploy: Used to automate CI/CD for backend code updates.

My Coding Process

I wrote modular Lambda functions using ES6, implemented security best practices like scoped IAM roles, and used Postman collections to test API responses. I used CloudWatch Logs and Alarms for debugging and monitoring. The pipeline automatically runs tests and deploys updates to production.

Accessibility in My Coding Process

Although this project is backend-focused, I ensured that any frontend components (admin dashboard or receipt pages) adhered to WCAG standards. I ran Lighthouse and used screen readers and keyboard testing for any public UI pages.
